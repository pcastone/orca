Examples

Real-world usage examples for Orca, Orchestrator, and ACO

Orca Examples

1. Create and Run a Simple Task

# Initialize Orca
orca init

# Create a simple Q&A task
orca task create "What is the capital of France?"

# List all tasks
orca task list

# Run the task (replace <task-id> with actual ID from list)
orca task run 1

# View task details
orca task get 1

2. Code Analysis Task

# Create a code analysis task using ReAct pattern (default)
orca task create "Analyze the src/ directory and list all Rust files"

# Run the task
orca task run 2

# Create a more complex analysis with Plan-Execute pattern
# First create the task, then set metadata
orca task create "Analyze the codebase structure and identify areas needing tests"
# Set pattern via metadata: {"pattern": "plan_execute"}

3. Code Generation with Reflection

# Create a code generation task with Reflection pattern
# for high-quality output
orca task create "Write a well-tested Rust function to parse JSON config files"
# Set pattern via metadata: {"pattern": "reflection"}

# The reflection pattern will:
# 1. Generate initial code
# 2. Critique the code
# 3. Refine based on critique
# 4. Repeat until quality threshold met

4. Enable Token Streaming

Edit ~/.orca/orca.toml:

[llm]
provider = "anthropic"
model = "claude-3-sonnet"
api_key = "${ANTHROPIC_API_KEY}"

[execution]
streaming = true  # Enable real-time token streaming

Orchestrator Examples

1. Setup and Start Orchestrator

# Navigate to orchestrator directory
cd src/crates/orchestrator

# Set up database
export DATABASE_URL="sqlite:orchestrator.db"
sqlx database create
sqlx migrate run

# Start the orchestrator server
cargo run --release

# Server will listen on http://localhost:50051

2. Docker Deployment

# Build Docker image
docker build -t orchestrator:latest .

# Run with environment variables
docker run -d \
  --name orchestrator \
  -p 50051:50051 \
  -e DATABASE_URL=sqlite:/data/orchestrator.db \
  -e ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY} \
  -v orchestrator-data:/data \
  orchestrator:latest

# Check logs
docker logs -f orchestrator

3. Multi-Machine Production Setup

# Machine 1: Orchestrator Server
export ORCHESTRATOR_HOST="0.0.0.0"
export ORCHESTRATOR_PORT="50051"
export DATABASE_URL="sqlite:/var/lib/orchestrator/db.sqlite"
cargo run -p orchestrator --release

# Machine 2: ACO Client
export ACO_SERVER="http://orchestrator-machine:50051"
aco tui

# Machine 3: Another ACO Client
export ACO_SERVER="http://orchestrator-machine:50051"
aco --connect admin:password tui

ACO Examples

1. Connect to Local Orchestrator

# Start local orchestrator first
cd src/crates/orchestrator
cargo run --release &

# In another terminal, start ACO TUI
aco tui

# Or specify server explicitly
aco --server http://localhost:50051 tui

2. Connect to Remote Orchestrator

# Connect to production server
aco --server https://orchestrator.example.com:50051 tui

# With authentication
aco --server https://orchestrator.example.com:50051 \
    --connect admin:password \
    tui

# Or use environment variables
export ACO_SERVER="https://orchestrator.example.com:50051"
export ACO_CONNECT="admin:password"
aco tui

3. Secure Access via SSH Tunnel

# Create SSH tunnel to remote server
ssh -L 50051:localhost:50051 user@remote-server &

# Connect ACO to local port (tunneled to remote)
aco --server http://localhost:50051 tui

# Now you have secure access to remote orchestrator

4. Complete TUI Workflow

# Start ACO TUI
aco tui

# Inside TUI:
# 1. Press '1' to view Tasks List
# 2. Use ↓/↑ or j/k to navigate tasks
# 3. Press Enter to view task details
# 4. Press 'e' to execute selected task
# 5. Press '3' to view Execution Stream
# 6. Watch real-time execution events
# 7. Press 'r' to refresh data
# 8. Press '?' for help
# 9. Press 'q' to quit

Configuration Examples

~/.orca/orca.toml

[llm]
provider = "anthropic"

Ollama (Local) Configuration

Project-Level Configuration

./.orca/orca.toml (overrides user config)

[execution]
streaming = true
max_retries = 3

[project]
workspace = "./workspace"

Workflow Examples

1. Multi-Task Workflow

# Create multiple related tasks
orca task create "List all Python files in the project"
orca task create "Analyze test coverage for Python files"
orca task create "Generate test report"

# Create workflow linking these tasks
orca workflow create "Python Analysis Pipeline"

# Add tasks to workflow (implementation depends on CLI)
# Run workflow
orca workflow run 1

2. Code Review Workflow

# Task 1: Analyze code quality
orca task create "Review src/main.rs for code quality issues"

# Task 2: Check for security vulnerabilities
orca task create "Scan codebase for security vulnerabilities"

# Task 3: Generate review report
orca task create "Compile code review findings into markdown report"

# Create and run workflow
orca workflow create "Code Review Pipeline"
orca workflow run <workflow-id>

3. Documentation Generation Workflow

# Using Reflection pattern for quality
orca task create "Analyze API endpoints and generate documentation"
# Set metadata: {"pattern": "reflection"}

orca task create "Review documentation for completeness"
# Set metadata: {"pattern": "reflection"}

orca task create "Generate final API documentation in markdown"
# Set metadata: {"pattern": "reflection"}

# Run as workflow
orca workflow create "API Documentation Generator"

Complete End-to-End Example

Here's a complete example from setup to execution:

# 1. Setup Orca
orca init

# 2. Configure LLM (edit ~/.orca/orca.toml)
cat > ~/.orca/orca.toml <<EOF
[llm]
provider = "anthropic"
model = "claude-3-sonnet"
api_key = "\${ANTHROPIC_API_KEY}"

[execution]
streaming = true
EOF

# 3. Set API key
export ANTHROPIC_API_KEY="your-api-key-here"

# 4. Create a task
orca task create "Analyze this Rust project and suggest improvements"

# 5. Run the task
orca task run 1

# 6. View results
orca task get 1

# 7. For distributed setup, start orchestrator
cd src/crates/orchestrator
cargo run --release &

# 8. Connect ACO TUI to monitor
aco tui

# 9. Execute task from TUI
# Press '1' for tasks, navigate with j/k, press 'e' to execute

# 10. View execution stream
# Press '3' to watch real-time execution events
