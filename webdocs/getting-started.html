Getting Started

Set up your development environment and build your first AI agent workflow

Prerequisites

Required Software

Rust 1.75.0 or higher

# Install via rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Verify installation
rustc --version
cargo --version

Git (for version control)

# Verify git is installed
git --version

Optional: Install cargo-watch for auto-rebuild and cargo-nextest for faster test execution

Installation

Step 1: Clone the Repository

git clone <repository-url>
cd orca

Step 2: Build Orca

Orca is the standalone orchestrator and the best starting point for new users.

# Option 1: Using the build script (recommended)
./scripts/build-orca.sh --install

# Option 2: Using cargo directly
cargo build -p orca --release

# Option 3: Install globally
cargo install --path src/crates/orca

Step 3: Initialize Orca

# Initialize Orca (creates ~/.orca/ directory)
orca init

This creates:

  • ~/.orca/ directory
  • ~/.orca/orca.db SQLite database
  • ~/.orca/orca.toml configuration file

Configuration

Configure LLM Provider

Edit ~/.orca/orca.toml:

[llm]
provider = "anthropic"          # Options: anthropic, openai, ollama, gemini, etc.
model = "claude-3-sonnet"       # Model name for your provider
api_key = "${ANTHROPIC_API_KEY}" # Reads from environment variable

[execution]
streaming = true                 # Enable real-time token streaming

Set Environment Variable

# For Anthropic Claude
export ANTHROPIC_API_KEY="your-api-key-here"

# For OpenAI
export OPENAI_API_KEY="your-api-key-here"

# For Google Gemini
export GOOGLE_API_KEY="your-api-key-here"

# Make permanent (add to ~/.bashrc or ~/.zshrc)
echo 'export ANTHROPIC_API_KEY="your-key"' >> ~/.bashrc
source ~/.bashrc

Never commit API keys to version control. Always use environment variables.

Your First Task

Create a Simple Task

# Create a task
orca task create "What is 2+2?"

# List all tasks
orca task list

# Run a specific task
orca task run <task-id>

# View task details
orca task get <task-id>

Using Different Agent Patterns

Orca supports three agent patterns:

ReAct (Default) - Best for Most Tasks (90%)

Quick responses for general Q&A and tool-using tasks

# Uses ReAct by default
orca task create "List all Python files in src/"

Plan-Execute - For Complex Multi-Step Tasks

Creates explicit plan before execution, ideal for research and analysis

# Create task with plan_execute pattern
orca task create "Analyze codebase and identify test coverage gaps"
# Set metadata: {"pattern": "plan_execute"}

Reflection - For Quality-Critical Output

Generate → critique → refine loop for high-quality results

# Create task with reflection pattern
orca task create "Write production-ready sorting function with tests"
# Set metadata: {"pattern": "reflection"}

Troubleshooting

"command not found: cargo"

Install Rust via rustup:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
Build fails with linking errors

Install build tools:

# macOS
xcode-select --install

# Linux
sudo apt-get install build-essential
Slow compilation times

Use more parallel jobs:

export CARGO_BUILD_JOBS=8
cargo build --release
