ACO

Terminal UI Client for Orchestrator

Overview

ACO is a powerful terminal user interface (TUI) for managing tasks and workflows in the Orca orchestration system. Built with Ratatui and Crossterm, it provides a responsive, feature-rich terminal experience for monitoring and executing AI agent workflows.

ACO connects to the Orchestrator via WebSocket for real-time task execution and monitoring, making it perfect for both local development and production environments.

Features

Beautiful TUI Interface

Responsive terminal interface built with Ratatui

Real-time Monitoring

Live execution streaming with color-coded events

Comprehensive Keyboard Navigation

Vim-style keys and intuitive shortcuts

Status Indicators

Color-coded icons and badges for instant recognition

Auto-refresh

Automatic data updates every 10 seconds

Remote Execution

Execute tasks on remote orchestrator servers

Installation

Build from Source

# Clone repository
git clone <repository-url>
cd orca

# Build release binary
cargo build -p aco --release

# Binary location
target/release/aco

Install Globally

# Install for current user
mkdir -p ~/.local/bin
cp target/release/aco ~/.local/bin/
export PATH="$HOME/.local/bin:$PATH"

# Or install globally (Linux/macOS)
sudo cp target/release/aco /usr/local/bin/

Verify Installation

aco --version
aco --help

Usage

Starting the TUI

# Use default server (localhost:50051)
aco tui

# Connect to specific server
aco --server http://example.com:50051 tui

# With authentication
aco --connect user:password tui

# Verbose mode
aco -v tui

Environment Variables

# Default server URL
export ACO_SERVER=http://localhost:50051

# Authentication string
export ACO_CONNECT=user:password

# Logging level
export RUST_LOG=info  # Options: debug, info, warn, error

Deployment

Docker Deployment

# Build image
docker build -t aco-tui:latest .

# Run with server connection
docker run -it --rm \
  -e ACO_SERVER=http://orchestrator:50051 \
  -e RUST_LOG=info \
  aco-tui:latest

# Run with authentication
docker run -it --rm \
  -e ACO_SERVER=http://orchestrator:50051 \
  -e ACO_CONNECT=user:password \
  aco-tui:latest

Systemd Service

Create /etc/systemd/system/aco-tui.service:

[Unit]
Description=ACO TUI Service
After=network.target orchestrator.service
Requires=orchestrator.service

[Service]
Type=forking
User=aco
Group=aco
Environment="ACO_SERVER=http://localhost:50051"
Environment="RUST_LOG=info"

# Start TUI in tmux session
ExecStart=/usr/bin/tmux new-session -d -s aco-tui '/usr/local/bin/aco tui'
ExecStop=/usr/bin/tmux kill-session -t aco-tui

Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable aco-tui
sudo systemctl start aco-tui

# Attach to TUI
sudo -u aco tmux attach -t aco-tui

Configuration

Server Connection

ACO connects to the orchestrator server via WebSocket. Ensure orchestrator is running:

# Start orchestrator
cargo run -p orchestrator --release

# In another terminal, start ACO
aco --server http://localhost:50051 tui

Security

Use TLS for production

export ACO_SERVER=https://orchestrator.example.com:50051

Use SSH tunnel for remote access

ssh -L 50051:localhost:50051 user@remote-server &
aco --server http://localhost:50051 tui
