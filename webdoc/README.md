# rLangGraph Documentation Website

Modern, responsive documentation website for rLangGraph built with Tailwind CSS.

## Overview

This is a static documentation website that showcases rLangGraph (Rust implementation of LangGraph), including comprehensive API documentation, guides, examples, and architecture details.

## Technology Stack

- **HTML5** - Semantic markup
- **Tailwind CSS 3.4** - Utility-first CSS framework
- **JavaScript (Vanilla)** - Interactive components
- **Node.js** - Build tooling
- **PostCSS** - CSS processing with Autoprefixer
- **Prism.js** - Syntax highlighting

## Quick Start

### 1. Install Dependencies

```bash
cd webdoc
npm install
```

This will install:
- `tailwindcss` - CSS framework
- `postcss` and `postcss-cli` - CSS processing
- `autoprefixer` - Browser compatibility
- `@tailwindcss/typography` - Typography plugin
- `prismjs` - Syntax highlighting

### 2. Build CSS

```bash
# Development build (watch mode)
npm run dev

# Production build (optimized)
npm run build
```

The build process:
1. Reads `src/css/main.css` (contains Tailwind directives)
2. Processes with PostCSS and Tailwind
3. Outputs to `dist/css/styles.css` (used by HTML files)

### 3. Serve Locally

```bash
# Using Python 3 (simple HTTP server)
npm run serve

# Or manually:
python3 -m http.server 8000
```

Then open http://localhost:8000 in your browser.

## Build Scripts

Defined in `package.json`:

- **`npm run dev`** - Watch mode for development. Auto-rebuilds CSS on file changes.
- **`npm run build`** - Production build with CSS optimization and purging.
- **`npm run serve`** - Start local HTTP server on port 8000.

## Adding New Pages

### Method 1: Copy Existing Template

1. Copy an existing page (e.g., `pages/architecture/index.html`)
2. Update the content between `<main>` tags
3. Update metadata in `<head>` (title, description, keywords)
4. Update navigation active state
5. Update breadcrumbs

### Method 2: Create from Scratch

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Your page description">
  <title>Your Page Title - rLangGraph</title>
  <link rel="stylesheet" href="../../dist/css/styles.css">
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/prism/1.29.0/themes/prism-tomorrow.min.css">
</head>
<body>
  <!-- Copy header from existing page -->
  <!-- Your content here -->
  <!-- Copy footer from existing page -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/prism/1.29.0/prism.min.js"></script>
  <script src="../../src/js/navigation.js"></script>
  <script src="../../src/js/search.js"></script>
</body>
</html>
```

### Important Notes

1. **CSS Path**: Adjust `dist/css/styles.css` path based on page location
2. **JS Path**: Adjust `src/js/*.js` paths based on page location
3. **Navigation**: Update active state on nav links
4. **Metadata**: Always update title, description, and keywords

## Customization

### Colors

Edit `tailwind.config.js`:

```javascript
theme: {
  extend: {
    colors: {
      primary: {
        // Your custom colors
      }
    }
  }
}
```

Then rebuild: `npm run build`

### Typography

The `@tailwindcss/typography` plugin is configured in `tailwind.config.js`. Customize prose styles there.

### Layout

Modify `src/css/main.css` for custom components and utilities.

## Performance

### Optimization Tips

1. **CSS Purging**: Production build automatically purges unused CSS
2. **CDN Resources**: Prism.js and fonts loaded from CDN (cached)
3. **Minimal JavaScript**: Only essential JS for navigation and search
4. **Static Assets**: All HTML is static (fast loading)

### Build Size

- Development CSS: ~3-4 MB (all Tailwind utilities)
- Production CSS: ~10-20 KB (purged, minified)

## Troubleshooting

### CSS not loading
- Ensure `npm run build` completed successfully
- Check `dist/css/styles.css` exists
- Verify CSS path in HTML is correct

### JavaScript errors
- Check browser console for errors
- Ensure Prism.js CDN is accessible
- Verify JS file paths are correct

### Search not working
- Check `src/js/search.js` is loaded
- Ensure search input has `id="search-input"`
- Check browser console for errors

### Build fails
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
npm run build
```
