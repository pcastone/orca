Simple State Graph

Beginner

Learn the fundamentals of StateGraph construction with nodes, edges, and state reducers. Master the core building blocks of rLangGraph.

10 min read

Overview

The StateGraph is the foundation of rLangGraph. It defines how data flows through your application using nodes (processing steps) and edges (connections).

Building Blocks

1. State

A struct that holds all data flowing through the graph. Must implement Clone, Serialize, Deserialize. 

#[derive(Clone, Serialize, Deserialize)]
struct MyState {
    counter: i32,
    messages: Vec<String>,
}

2. Nodes

Async functions that take state and return modified state. Each node processes data. 

async fn my_node(mut state: MyState) -> Result<MyState, GraphError> {
    state.counter += 1;
    Ok(state)
}

3. Edges

Connections between nodes that define execution order. 

graph.add_edge("node_a", "node_b");  // node_a → node_b

Example 1: Linear Flow

A simple graph where nodes execute in sequence:

use langgraph_core::{StateGraph, GraphError};
use serde::{Deserialize, Serialize};

#[derive(Clone, Serialize, Deserialize)]
struct State {
    value: i32,
}

async fn multiply_by_two(mut state: State) -> Result<State, GraphError> {
    state.value *= 2;
    println!("After multiply: {}", state.value);
    Ok(state)
}

async fn add_ten(mut state: State) -> Result<State, GraphError> {
    state.value += 10;
    println!("After add: {}", state.value);
    Ok(state)
}

#[tokio::main]
async fn main() -> Result<(), GraphError> {
    let mut graph = StateGraph::new();

    // Add nodes
    graph.add_node("multiply", multiply_by_two);
    graph.add_node("add", add_ten);

    // Define flow: start → multiply → add → end
    graph.add_edge("__start__", "multiply");
    graph.add_edge("multiply", "add");
    graph.add_edge("add", "__end__");

    // Compile
    let compiled = graph.compile()?;

    // Execute
    let result = compiled.invoke(State { value: 5 }).await?;
    println!("Final: {}", result.value);  // (5 * 2) + 10 = 20

    Ok(())
}

Output:

After multiply: 10
After add: 20
Final: 20

Example 2: Parallel Nodes

Multiple nodes executing in the same superstep:

#[derive(Clone, Serialize, Deserialize)]
struct ParallelState {
    results: Vec<String>,
}

async fn worker_a(mut state: ParallelState) -> Result<ParallelState, GraphError> {
    state.results.push("Worker A done".to_string());
    Ok(state)
}

async fn worker_b(mut state: ParallelState) -> Result<ParallelState, GraphError> {
    state.results.push("Worker B done".to_string());
    Ok(state)
}

async fn aggregator(state: ParallelState) -> Result<ParallelState, GraphError> {
    println!("Aggregating {} results", state.results.len());
    Ok(state)
}

#[tokio::main]
async fn main() -> Result<(), GraphError> {
    let mut graph = StateGraph::new();

    graph.add_node("worker_a", worker_a);
    graph.add_node("worker_b", worker_b);
    graph.add_node("aggregate", aggregator);

    // Both workers execute in parallel
    graph.add_edge("__start__", "worker_a");
    graph.add_edge("__start__", "worker_b");

    // Aggregator waits for both
    graph.add_edge("worker_a", "aggregate");
    graph.add_edge("worker_b", "aggregate");
    graph.add_edge("aggregate", "__end__");

    let compiled = graph.compile()?;
    let result = compiled.invoke(ParallelState {
        results: vec![]
    }).await?;

    for r in &result.results {
        println!("- {}", r);
    }

    Ok(())
}

Example 3: State Reducers

Understanding how state changes are merged:

#[derive(Clone, Serialize, Deserialize)]
struct CounterState {
    count: i32,          // OverwriteReducer
    messages: Vec<String>,  // AppendReducer
}

async fn increment_a(mut state: CounterState) -> Result<CounterState, GraphError> {
    state.count = 1;
    state.messages.push("A incremented".to_string());
    Ok(state)
}

async fn increment_b(mut state: CounterState) -> Result<CounterState, GraphError> {
    state.count = 2;
    state.messages.push("B incremented".to_string());
    Ok(state)
}

#[tokio::main]
async fn main() -> Result<(), GraphError> {
    let mut graph = StateGraph::new();

    graph.add_node("inc_a", increment_a);
    graph.add_node("inc_b", increment_b);

    // Both run in parallel
    graph.add_edge("__start__", "inc_a");
    graph.add_edge("__start__", "inc_b");
    graph.add_edge("inc_a", "__end__");
    graph.add_edge("inc_b", "__end__");

    let compiled = graph.compile()?;
    let result = compiled.invoke(CounterState {
        count: 0,
        messages: vec![],
    }).await?;

    // count: Last write wins (2)
    // messages: Both appended ["A incremented", "B incremented"]
    println!("Count: {}", result.count);
    println!("Messages: {:?}", result.messages);

    Ok(())
}

Reducer Behavior:

  • i32, String, etc.: Last write wins (OverwriteReducer)
  • Vec<T>: All items appended (AppendReducer)
  • HashMap<K,V>: Deep merge (MergeReducer)
