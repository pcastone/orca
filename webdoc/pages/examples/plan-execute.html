Plan-Execute Pattern

Intermediate

Build a planning agent that breaks down complex tasks into steps, executes them, and replans based on results.

Overview

The Plan-Execute pattern is ideal for complex, multi-step tasks where upfront planning improves execution quality. The agent creates a plan, executes each step, and replans if needed based on results.

Pattern Flow:

1. Plan: LLM breaks task into numbered steps
2. Execute: Execute first incomplete step with tools
3. Observe: Check if task is complete
4. Replan (if needed): Adjust plan based on results
5. Repeat: Until all steps complete

Complete Example

use langgraph_core::{StateGraph, Message, Tool, GraphError};
use serde::{Deserialize, Serialize};

#[derive(Clone, Serialize, Deserialize)]
struct PlanExecuteState {
    messages: Vec<Message>,
    plan: Vec<String>,
    completed_steps: Vec<usize>,
    current_step: Option<usize>,
}

// Planner node - creates the initial plan
async fn planner(
    mut state: PlanExecuteState,
    llm: &LlmClient,
) -> Result<PlanExecuteState, GraphError> {
    let prompt = format!(
        "Break down this task into a numbered step-by-step plan:\n{}",
        state.messages.last().unwrap().content()
    );

    let response = llm.chat(&[Message::system(
        "You are a planning assistant. Create clear, numbered steps."
    ), Message::human(prompt)]).await?;

    // Parse plan from response (simplified)
    let plan: Vec<String> = response
        .lines()
        .filter(|l| l.trim().starts_with(char::is_numeric))
        .map(|s| s.to_string())
        .collect();

    state.plan = plan;
    state.current_step = Some(0);
    state.messages.push(Message::assistant(response));

    Ok(state)
}

// Executor node - executes current step
async fn executor(
    mut state: PlanExecuteState,
    llm: &LlmClient,
    tools: &[Tool],
) -> Result<PlanExecuteState, GraphError> {
    let step_idx = state.current_step.unwrap();
    let step = &state.plan[step_idx];

    let prompt = format!("Execute this step: {}", step);
    let response = llm.chat_with_tools(&[
        Message::system("Execute the given step using available tools."),
        Message::human(prompt),
    ], tools).await?;

    state.messages.push(Message::assistant(response));
    state.completed_steps.push(step_idx);

    Ok(state)
}

// Replanner - adjusts plan if needed
async fn replanner(
    mut state: PlanExecuteState,
    llm: &LlmClient,
) -> Result<PlanExecuteState, GraphError> {
    let prompt = format!(
        "Current plan:\n{}\n\nCompleted: {:?}\n\nDo we need to adjust the plan?",
        state.plan.join("\n"),
        state.completed_steps
    );

    let response = llm.chat(&[Message::human(prompt)]).await?;

    // If LLM suggests changes, update plan (simplified)
    if response.contains("adjust") || response.contains("change") {
        // Parse new plan...
        println!("Replanning based on results");
    }

    Ok(state)
}

// Routing logic
fn should_continue(state: &PlanExecuteState) -> String {
    if state.completed_steps.len() >= state.plan.len() {
        "__end__".to_string()
    } else if state.completed_steps.len() % 3 == 0 && !state.completed_steps.is_empty() {
        "replan".to_string()  // Replan every 3 steps
    } else {
        "execute".to_string()
    }
}

#[tokio::main]
async fn main() -> Result<(), GraphError> {
    let llm = LlmClient::new(env::var("ANTHROPIC_API_KEY")?);
    let tools = create_tools();  // Your tools here

    let mut graph = StateGraph::new();

    graph.add_node("plan", |s| planner(s, &llm));
    graph.add_node("execute", |s| executor(s, &llm, &tools));
    graph.add_node("replan", |s| replanner(s, &llm));

    graph.add_edge("__start__", "plan");
    graph.add_conditional_edges("plan", should_continue);
    graph.add_conditional_edges("execute", should_continue);
    graph.add_conditional_edges("replan", should_continue);

    let compiled = graph.compile()?;

    let result = compiled.invoke(PlanExecuteState {
        messages: vec![Message::human(
            "Research and write a comprehensive report on renewable energy"
        )],
        plan: vec![],
        completed_steps: vec![],
        current_step: None,
    }).await?;

    println!("Completed {} steps", result.completed_steps.len());
    Ok(())
}

When to Use Plan-Execute

  • Complex tasks requiring 5+ distinct steps
  • Research projects where planning improves quality
  • Multi-tool workflows with dependencies between steps
  • Long-running tasks that benefit from structured approach

Trade-offs: Plan-Execute is slower than ReAct (extra planning step) but produces higher quality results for complex tasks. For simple questions, use ReAct instead.
