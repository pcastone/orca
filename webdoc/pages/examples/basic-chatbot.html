Basic Chatbot

Beginner

Build your first conversational AI agent with rLangGraph. This tutorial covers state management, message history, and basic LLM integration.

15 min read Rust

Overview

In this tutorial, you'll build a simple conversational chatbot using rLangGraph. The chatbot will:

  • Maintain conversation history across multiple turns
  • Use state management with the AppendReducer
  • Integrate with an LLM (Claude, OpenAI, or Ollama)
  • Handle user input and generate responses

What you'll learn: Core rLangGraph concepts including StateGraph, state reducers, message types, and basic LLM integration.

Prerequisites

Before starting, ensure you have:

  • Rust 1.75+ installed (rustc --version)
  • Cargo package manager
  • An API key for your chosen LLM provider:
    • Anthropic Claude: ANTHROPIC_API_KEY
    • OpenAI: OPENAI_API_KEY
    • Or a local Ollama instance running

Dependencies

Add these to your Cargo.toml:

[dependencies]
langgraph-core = "0.1"
langgraph-checkpoint = "0.1"
serde = { version = "1.0", features = ["derive"] }

# ollama-rs = "0.1"  # For Ollama

Complete Code

Here's the complete chatbot implementation. We'll break it down step-by-step in the next section. 

use langgraph_core::{StateGraph, Message, GraphError};
use langgraph_checkpoint::MemoryCheckpointSaver;
use serde::{Deserialize, Serialize};
use std::env;

// Define the chatbot state
#[derive(Clone, Debug, Serialize, Deserialize)]
struct ChatbotState {
    // Messages array with AppendReducer behavior
    messages: Vec<Message>,
}

// Chatbot node function
async fn chatbot_node(mut state: ChatbotState) -> Result<ChatbotState, GraphError> {
    // Get the last user message
    let last_message = state.messages.last()
        .ok_or_else(|| GraphError::InvalidState("No messages in state".to_string()))?;

    // Simple echo response (replace with actual LLM call)
    let response = format!("You said: {}", last_message.content());

    // Add assistant response to messages
    state.messages.push(Message::assistant(response));

    Ok(state)
}

// Build and run the chatbot
#[tokio::main]
async fn main() -> Result<(), GraphError> {
    // Create the state graph
    let mut graph = StateGraph::new();

    // Add the chatbot node
    graph.add_node("chatbot", chatbot_node);

    // Define edges: start -> chatbot -> end
    graph.add_edge("__start__", "chatbot");
    graph.add_edge("chatbot", "__end__");

    // Compile the graph with a checkpoint saver
    let checkpointer = MemoryCheckpointSaver::new();
    let compiled = graph.compile()
        .with_checkpointer(checkpointer);

    // Create initial state
    let initial_state = ChatbotState {
        messages: vec![
            Message::system("You are a helpful assistant."),
            Message::human("Hello! What can you help me with?"),
        ],
    };

    // Run the graph
    let result = compiled.invoke(initial_state).await?;

    // Print the conversation
    println!("Conversation:");
    for msg in &result.messages {
        match msg {
            Message::System(content) => println!("System: {}", content),
            Message::Human(content) => println!("User: {}", content),
            Message::Assistant(content) => println!("Assistant: {}", content),
            _ => {}
        }
    }

    Ok(())
}

Step-by-Step Breakdown

Step 1: Define the State

First, we define the chatbot's state structure. The messages field uses Vec<Message> which automatically uses the AppendReducer - new messages are appended to the existing array. 

#[derive(Clone, Debug, Serialize, Deserialize)]
struct ChatbotState {
    // Messages are accumulated across turns
    messages: Vec<Message>,
}

Note: The AppendReducer ensures that when multiple nodes add messages, they're all preserved in order. This is crucial for maintaining conversation history.

Step 2: Implement the Chatbot Node

The chatbot node receives the current state, processes the last message, and returns an updated state with the assistant's response. 

async fn chatbot_node(mut state: ChatbotState) -> Result<ChatbotState, GraphError> {
    // Get the last message (user input)
    let last_message = state.messages.last()
        .ok_or_else(|| GraphError::InvalidState("No messages".to_string()))?;

    // Generate response (simplified - use LLM in production)
    let response = format!("You said: {}", last_message.content());

    // Add response to message history
    state.messages.push(Message::assistant(response));

    Ok(state)
}

Step 3: Build the Graph

Create a StateGraph, add the chatbot node, and define the execution flow. 

let mut graph = StateGraph::new();

// Add node
graph.add_node("chatbot", chatbot_node);

// Define flow: start -> chatbot -> end
graph.add_edge("__start__", "chatbot");
graph.add_edge("chatbot", "__end__");

Step 4: Add Checkpointing

Compile the graph with a checkpoint saver to enable state persistence and resume capability. 

let checkpointer = MemoryCheckpointSaver::new();
let compiled = graph.compile()
    .with_checkpointer(checkpointer);

Step 5: Execute the Graph

Create the initial state with a system message and user input, then invoke the graph. 

let initial_state = ChatbotState {
    messages: vec![
        Message::system("You are a helpful assistant."),
        Message::human("Hello! What can you help me with?"),
    ],
};

let result = compiled.invoke(initial_state).await?;

Key Concepts

StateGraph

The core abstraction for building agent workflows. It manages nodes, edges, and state flow.

Message Types

rLangGraph supports multiple message types:

  • Message::system() - System instructions for the LLM
  • Message::human() - User input
  • Message::assistant() - LLM responses
  • Message::tool_call() - Tool invocations
  • Message::tool_result() - Tool results

State Reducers

Vec<Message> automatically uses AppendReducer, which combines messages from parallel nodes by appending them. This ensures no messages are lost during execution.

Checkpointing

The MemoryCheckpointSaver stores state snapshots after each step, enabling time-travel debugging and workflow resumption.

Common Issues

Graph compilation fails

Ensure all nodes have edges connecting them. Every node must be reachable from __start__ and must reach __end__.

State not updating

Make sure your node function returns the modified state. The state is immutable by default - you must return a new/modified copy.

Messages disappearing

Verify you're using Vec<Message> for the messages field. This ensures the AppendReducer is used automatically.
