rLangGraph

Build stateful, multi-actor applications with large language models using Rust. A port of Python's LangGraph with the Pregel execution model.

Get Started View Examples

Key Features

Stateful Execution

Persistent state across multiple execution steps with explicit state management and reducers.

Checkpoint & Resume

Save and restore execution state at any point. Supports time travel debugging and branching.

Human-in-the-Loop

Pause execution for human approval or input with interrupts and approval workflows.

Pregel Execution

Bulk Synchronous Parallel model with supersteps, barriers, and deterministic execution.

Streaming-First

Real-time event emission during execution with support for streaming LLM responses.

Type-Safe

Leverage Rust's type system for compile-time validation and prevention of many error classes.

Quick Start

Installation

# Add to your Cargo.toml
cargo add langgraph-core langgraph-checkpoint langgraph-prebuilt

Basic Example

use langgraph_core::{StateGraph, GraphError};
use serde::{Deserialize, Serialize};

#[derive(Clone, Serialize, Deserialize)]
struct AgentState {
    messages: Vec<String>,
    count: i32,
}

#[tokio::main]
async fn main() -> Result<(), GraphError> {
    let mut graph = StateGraph::new();

    // Add nodes
    graph.add_node("process", |mut state: AgentState| {
        Box::pin(async move {
            state.messages.push("Processed!".to_string());
            state.count += 1;
            Ok(state)
        })
    });

    // Add edges
    graph.add_edge("__start__", "process");
    graph.add_edge("process", "__end__");

    // Compile and execute
    let compiled = graph.compile()?;
    let initial_state = AgentState {
        messages: vec!["Hello".to_string()],
        count: 0,
    };

    let result = compiled.invoke(initial_state).await?;
    println!("Messages: {:?}, Count: {}", result.messages, result.count);
    Ok(())
}
